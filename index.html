Connect your database

Uses Supabase (free) to store your food logs permanently — synced across all your devices.

Step 1 — Create a free Supabase project
  1. Go to supabase.com → New project
  2. Open SQL Editor and run:
create table food_db ( id uuid primary key default gen_random_uuid(), name text unique not null, kcal numeric not null, prot numeric not null, carbs numeric not null default 0, fats numeric not null default 0, fibre numeric not null default 0, unit text not null default 'g' ); create table food_log ( id uuid primary key default gen_random_uuid(), log_date date not null, meal text not null, food_name text not null, qty numeric not null, created_at timestamptz default now() ); create table settings ( key text primary key, value text ); alter table food_db enable row level security; alter table food_log enable row level security; alter table settings enable row level security; create policy "allow all" on food_db for all using (true) with check (true); create policy "allow all" on food_log for all using (true) with check (true); create policy "allow all" on settings for all using (true) with check (true);
⚠ Existing DB? Run this migration first: alter table food_db add column if not exists carbs numeric not null default 0; alter table food_db add column if not exists fats numeric not null default 0; alter table food_db add column if not exists fibre numeric not null default 0; create table if not exists weight_log ( id uuid primary key default gen_random_uuid(), log_date date not null unique, weight_kg numeric not null, created_at timestamptz default now() ); alter table weight_log enable row level security; create policy "allow all" on weight_log for all using (true) with check (true);
Step 2 — Paste your credentials

In Supabase → Settings → API, copy your Project URL and anon key.
