Connect your database

This app uses Supabase (free) to store your food logs permanently — across devices, forever. Set it up once in about 5 minutes.

Step 1 — Create a free Supabase project
  1. Go to supabase.com → New project
  2. Once created, go to SQL Editor and run this:
create table food_db ( id uuid primary key default gen_random_uuid(), name text unique not null, kcal numeric not null, prot numeric not null, unit text not null default 'g' ); create table food_log ( id uuid primary key default gen_random_uuid(), log_date date not null, meal text not null, food_name text not null, qty numeric not null, created_at timestamptz default now() ); create table settings ( key text primary key, value text ); alter table food_db enable row level security; alter table food_log enable row level security; alter table settings enable row level security; create policy "allow all" on food_db for all using (true) with check (true); create policy "allow all" on food_log for all using (true) with check (true); create policy "allow all" on settings for all using (true) with check (true);
Step 2 — Paste your project credentials

In Supabase → Settings → API, copy your Project URL and anon public key.
